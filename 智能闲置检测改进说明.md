# 智能闲置检测改进说明

## 问题描述

用户反馈：闲置检测功能有问题，很轻易就闲置了，然后就不录制了。用户在思考的过程中也可能被判断为"闲置"。

## 问题分析

原有系统的局限性：
1. 闲置阈值太短（300秒 = 5分钟）
2. 仅依赖键盘/鼠标活动检测
3. 没有考虑用户的思考时间
4. 单一维度检测，容易误判

## 解决方案

### 1. 创建智能活跃度监测器 (SmartActivityMonitor)

**核心改进：**
- 多维度检测：键盘/鼠标 + 窗口焦点 + CPU使用率
- 滑动窗口检测：允许一定时间的思考间歇
- 自适应阈值：根据使用模式调整闲置判断
- 智能判断：结合多个因素判断是否真正闲置

**新功能特性：**
- 思考模式：在指定时间窗口内有最少活动次数，不算闲置
- CPU检测：CPU使用率超过阈值时自动判定为活跃
- 窗口检测：切换窗口也视为活跃，适合工作场景

### 2. 更新配置参数

**config.py 中的新配置：**

```python
# 活跃度感知配置
ENABLE_AUTO_PAUSE = True
IDLE_THRESHOLD = 900  # 闲置阈值（秒），默认15分钟（原300秒）
THINKING_WINDOW = 300  # 思考窗口（秒），默认5分钟
MIN_ACTIVITY_COUNT = 3  # 思考窗口内最少需要几次操作
CPU_THRESHOLD = 15.0  # CPU使用率阈值（%）
ENABLE_CPU_CHECK = True  # 是否启用CPU使用率检测
ENABLE_WINDOW_CHECK = True  # 是否启用窗口焦点检测
AUTO_PAUSE_THRESHOLD = 120  # 自动暂停阈值（秒）
AUTO_RESUME_THRESHOLD = 10  # 自动恢复阈值（秒）
```

### 3. 集成到录制系统

已更新 `core/recorder.py`：
- 使用新的 `SmartAutoPauseRecorder` 替代旧的 `AutoPauseRecorder`
- 传递新的配置参数
- 增强日志输出，显示详细的配置信息

## 关键改进点

### 1. 闲置阈值延长
- **旧配置**: 300秒（5分钟）
- **新配置**: 900秒（15分钟）
- **好处**: 更宽容，不会轻易误判为闲置

### 2. 思考模式
- **思考窗口**: 300秒（5分钟）
- **最少活动次数**: 3次
- **工作原理**: 在5分钟内如果有3次操作，就算处于思考模式，不会判定为闲置
- **好处**: 允许用户有思考时间，不会误判思考为闲置

### 3. 多维度检测
- **键盘/鼠标**: 基础检测
- **窗口焦点**: 切换窗口视为活跃
- **CPU使用率**: 超过15%视为活跃
- **好处**: 即使不操作鼠标键盘，只要有窗口切换或高CPU使用，也能判定为活跃

### 4. 自适应判断
- 结合活动次数、闲置时间、思考模式、CPU使用率等多个因素
- 更加智能和准确的闲置判断

## 使用建议

1. **闲置阈值**: 根据个人工作习惯调整，建议10-20分钟
2. **思考窗口**: 建议3-5分钟，允许适度的思考时间
3. **最少活动次数**: 建议2-5次，取决于思考窗口长度
4. **CPU阈值**: 建议10-20%，根据机器性能调整
5. **窗口检测**: 工作场景建议启用，游戏场景可关闭
